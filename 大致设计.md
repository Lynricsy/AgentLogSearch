

AgentLogSearch 项目开发规格书

0. 项目一句话

开发一个基于 Next.js + Tailwind CSS + HeroUI + NestJS + PostgreSQL/pgvector 的 Agent CLI 对话历史语义检索系统。系统定时扫描本地 Agent CLI 历史记录，解析为统一会话模型，切分为语义 chunk，生成 embedding 后写入 pgvector，支持自然语言语义搜索，并能回溯完整对话、工作目录、thread id 和 resume 命令。

HeroUI 是基于 Tailwind CSS 的 React UI 组件库，适合 Next.js 前端；pgvector 是 PostgreSQL 的向量扩展，支持向量存储、精确/近似最近邻搜索、cosine/L2/inner product 等距离计算。

5.3 MockProvider

用于开发，不追求效果，只保证流程能跑。

export class MockEmbeddingProvider implements EmbeddingProvider {
  name = 'mock-1024';
  dimension = 1024;
  async embedTexts(texts: string[]): Promise<number[][]> {
    return texts.map((text) => {
      const arr = new Array(1024).fill(0);
      for (let i = 0; i < text.length; i++) {
        arr[i % 1024] += text.charCodeAt(i) / 10000;
      }
      return arr;
    });
  }
}

6. 后端核心流程

6.1 扫描流程

ScannerScheduler 定时触发
        ↓
ScannerService 查询 enabled=true 的 agent_source
        ↓
遍历 root_path 下文件
        ↓
计算 fingerprint：path + size + modified_at + hash
        ↓
判断是否新增或变更
        ↓
ParserRegistry 根据 source.type 选择 parser
        ↓
解析为 ParsedSession[]
        ↓
upsert agent_session
        ↓
删除旧 agent_message / agent_chunk
        ↓
插入新 agent_message
        ↓
ChunkerService 生成 chunk
        ↓
插入 agent_chunk，embedding_status=PENDING
        ↓
更新 scan_job

⸻

6.2 向量化流程

EmbeddingWorker 定时触发
        ↓
查询 embedding_status=PENDING 的 chunk
        ↓
标记 PROCESSING
        ↓
调用 EmbeddingProvider
        ↓
写入 embedding
        ↓
标记 READY
        ↓
失败则标记 FAILED 并记录错误

⸻

6.3 搜索流程

用户输入 query
        ↓
SearchController 接收 POST /search/semantic
        ↓
EmbeddingProvider 对 query 生成向量
        ↓
SearchService 使用 pgvector 查询 topK chunks
        ↓
按 session_id 聚合
        ↓
查询 session 元数据
        ↓
返回 session 级搜索结果

10. 向量搜索 SQL

10.1 基础语义搜索

SELECT
    c.id AS chunk_id,
    c.session_id,
    c.chunk_text,
    c.cwd,
    c.external_thread_id,
    c.agent_name,
    c.start_message_seq,
    c.end_message_seq,
    1 - (c.embedding <=> $1::vector) AS similarity
FROM agent_chunk c
WHERE c.embedding_status = 'READY'
  AND c.embedding IS NOT NULL
ORDER BY c.embedding <=> $1::vector
LIMIT $2;

pgvector 查询要按距离操作符排序并设置 LIMIT，这样才能有效使用向量索引；cosine distance 操作符是 <=>。 

⸻

10.2 带过滤条件

SELECT
    c.id AS chunk_id,
    c.session_id,
    c.chunk_text,
    c.cwd,
    c.external_thread_id,
    c.agent_name,
    c.start_message_seq,
    c.end_message_seq,
    1 - (c.embedding <=> $1::vector) AS similarity
FROM agent_chunk c
WHERE c.embedding_status = 'READY'
  AND c.embedding IS NOT NULL
  AND ($2::text IS NULL OR c.agent_name = $2)
  AND ($3::text IS NULL OR c.cwd ILIKE '%' || $3 || '%')
ORDER BY c.embedding <=> $1::vector
LIMIT $4;

10.3 后端聚合逻辑

伪代码：

const chunks = await vectorSearch(queryEmbedding, topK);
const groups = new Map<number, AggregatedSession>();
for (const chunk of chunks) {
  const group = groups.get(chunk.sessionId) ?? createGroup(chunk);
  group.score = Math.max(group.score, chunk.similarity);
  group.matchedChunks.push(chunk);
  groups.set(chunk.sessionId, group);
}
const results = [...groups.values()]
  .map((g) => ({
    ...g,
    matchedChunks: g.matchedChunks
      .sort((a, b) => b.similarity - a.similarity)
      .slice(0, 3),
  }))
  .sort((a, b) => b.score - a.score)
  .slice(0, sessionLimit);

const API_BASE_URL =
  process.env.NEXT_PUBLIC_API_BASE_URL ?? 'http://localhost:3001/api';
export async function semanticSearch(payload: SemanticSearchRequest) {
  const res = await fetch(`${API_BASE_URL}/search/semantic`, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
    },
    body: JSON.stringify(payload),
  });
  if (!res.ok) {
    throw new Error('Semantic search failed');
  }
  return res.json() as Promise<SemanticSearchResponse>;
}

18. Agent 开发提示词

可以直接给开发 Agent 这段：

你要开发一个 monorepo 项目 AgentLogSearch。
技术栈：
- 前端：Next.js App Router + TypeScript + Tailwind CSS + HeroUI
- 后端：NestJS + TypeScript
- 数据库：PostgreSQL + pgvector
- 数据库访问：Prisma 负责普通 CRUD，pg 原生 SQL 负责 pgvector 查询
- 部署：docker-compose
核心功能：
1. 配置 Agent CLI 历史源 agent_source
2. 扫描本地历史文件
3. 支持 JSONL / JSON / Markdown parser
4. 解析 session/message/cwd/threadId
5. 将 message 切成 agent_chunk
6. 使用 embedding provider 生成向量
7. 写入 pgvector
8. 支持语义搜索
9. 搜索结果按 session 聚合
10. 展示 matched chunks、cwd、thread id、resume command
11. 支持查看完整会话详情
12. 支持复制 resume command
优先开发顺序：
1. docker-compose + 数据库 migration
2. NestJS 基础模块和 sources CRUD
3. scanner + jsonl parser
4. chunker + mock embedding
5. pgvector semantic search API
6. Next.js 搜索页和详情页
7. sources 页面和 scan-jobs 页面
注意：
- 系统只生成 resume command，不执行本地命令。
- embedding 先实现 mock provider，保证全流程跑通。
- pgvector 查询使用 cosine distance：ORDER BY embedding <=> queryVector LIMIT topK。
- 搜索返回的是 session 级结果，不是裸 chunk 结果。

19. 课设答辩亮点

可以总结成：

1. 数据库不是普通 CRUD，而是关系数据 + 向量数据混合建模
2. 使用 pgvector 实现语义检索
3. 使用 chunk 切分避免长会话语义稀释
4. 使用异步 embedding worker 避免扫描阻塞
5. 使用 parser 适配器支持多种 Agent CLI 历史格式
6. 搜索结果能回溯完整 thread、cwd 和 resume 命令
7. 前后端分离，具备完整工程闭环

最终项目名建议：

AgentLogSearch：基于 pgvector 的 Agent CLI 对话历史语义检索系统
